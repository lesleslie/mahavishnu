## Data Validation Pipeline

Use this tool to validate structured data with schema checks, quality rules, profiling, and monitoring.

## Focus areas

- Schema and type validation
- Data quality and profiling
- Cross-field and business-rule checks
- Error handling and reporting
- Streaming or batch validation

## Workflow

1. Define the schema and rule set.
1. Check quality, integrity, and temporal consistency.
1. Decide how to handle partial failures.
1. Add metrics and alerts for validation drift.
1. Keep the validation logic extensible and deterministic.

## Output

- Validation rules and schema notes
- Error/reporting checklist
- Monitoring and alerting guidance
