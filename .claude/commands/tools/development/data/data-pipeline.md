## Data Pipeline Architecture

Use this tool to design a scalable data pipeline with ingestion, transformation, orchestration, storage, and quality controls.

## Focus areas

- Ingestion from APIs, databases, files, and streams
- Transformation and feature engineering
- Orchestration and retries
- Storage, partitioning, and retention
- Streaming, quality, governance, and scale

## Workflow

1. Identify source types, target storage, and freshness needs.
1. Choose batch, streaming, or hybrid processing.
1. Add validation and failure handling early.
1. Design orchestration, monitoring, and scaling around the data shape.
1. Keep cloud-specific details as implementation notes, not the core prompt.

## Output

- Pipeline architecture recommendation
- Ingestion/transformation/orchestration checklist
- Monitoring and governance notes
