## Code Explanation and Analysis

Use this tool to explain complex code clearly and progressively.

## Focus areas

- Code comprehension and complexity assessment
- Narrative explanation of control flow and concepts
- Visual aids and diagrams when useful
- Layered explanations for different skill levels

## Workflow

1. Analyze the code structure and identify the main concepts.
1. Explain the code from the top level down to the details.
1. Add diagrams or examples only where they improve clarity.
1. Call out tradeoffs, patterns, and dependencies.
1. Keep the explanation concise and readable.

## Output

- Plain-language explanation
- Diagram or call-flow notes
- Key concepts and takeaways
