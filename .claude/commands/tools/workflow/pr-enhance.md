## Pull Request Enhancement

Use this tool to make pull requests easier to review, safer to ship, and clearer to understand.

## Focus areas

- Change summary and impact analysis
- Review-friendly organization
- Test and documentation evidence
- Security, approval, and communication notes

## Workflow

1. Summarize the change set and likely impact.
1. Group files and call out risks or dependencies.
1. Add testing, validation, and rollout context.
1. Include security and approval notes when relevant.
1. Keep the description concise enough to review quickly.

## Output

- PR summary and impact notes
- Review checklist
- Validation and communication points
